6月 2014
产品设计

6月11日

今天老倪组织的第一场会议上，终于提出了融合团队的思路，我是举双手双脚赞成的，不过就我们自己按项目来融合团队，其实有几个点是必须注意的：

  1. 团队规范问题，就是敏捷流程如何执行的问题，这个是需要scrum master来落实和推动的；
  2. 产品方向问题，当前的产品缺少规划的主要原因是上面的目标不明确，实际需求来源仅仅是产品或领导的拍脑袋；
  3. 产品内部质量问题，就是设计质量和编码质量，从项目管理上来说，”时间、成本、质量“的三角关系，我们只是拿时间做接口，而放弃了对质量的把控；
  4. 公平性问题，如何公正的衡量成员对团队所付出的，而不是不管加班在做什么，仅仅是为了晒个朋友圈给老板看（老宋，不是洗你哈，只是觉得现在下面做事的都是这种心态，不好）；
  5. 还有很多，不想写了，搞的不像是工作日报了。。。。

今天老倪的第二场会，其实是一个迟到的会议，上周就应该过的，还是有些东西要说一下：

  1. 设计的态度是，这是在完成老倪布置的任务；
  2. 老倪说ok了，就直接交给开发执行；
  3. 不让说流程，那就拿我所待过的几个软件公司的情况举例吧，产品规划功能（调研及数据支撑根据规模和阶段，在两周到两个月不等）－> ue设计线框细化用户体验并交由团队评审（线框保真度中等，中级人员平均 4页/人天）－> ui设计提供设计方案（初级人员平均每人一天两到三套方案）－>ui根据敲定的方案制作效果图并交产品评测（初级人员平均每人一天5个页面，无素材的情况下一天至少两个页面）－>测试和开发根据线框模型及效果图进入开发阶段（中级程序（c#） 50行/人天 ，不包括ut测试代码及系统自动生成的代码）
  4. 随手一写，妈的又写多了。。。只是想估算下大家当前的效率。
  5. 当然，对执行层面的要求不要太高，并不是每个人都是以做事业的心态来工作的。人性本懒惰，好逸恶劳是通病，这点，理解但不赞同。
  6. 说到这儿，我希望的团队，是能有共同价值观一起做事，一起奋斗的，如果团队决定了，你可以不认可我们，但请一定支持我们。
